Installation / Patching

UO Outlands

  1. Download GumpOverrides.
  2. Unzip into a GumpOverrides folder inside your Outlands folder.

Classic Shards

  1. Download UO_ART_MODS.zip.
  2. Extract the downloaded zip to a new folder anywhere.
  3. Use UOFiddler (recommended) to handle the full install via its MassImport plugin.
  4. Create a backup of your Ultima Online client folder.
  5. Watch the install video below (unmute), or download the install guide video.

Setting up UOFiddler

  1. Download the latest release of UOFiddler.
  2. Settings > Path Settings → paste your Ultima folder path → click Reload paths (list should populate with .mul, .idx, and .uop files).
  3. Setting > Options > Output Path → set to a new local OUTPUT folder (this is where the .mul and .uop files will be written).
  4. Plugins > Manage → enable MassImportPlugin.
  5. Plugins > Manage → enable UOPpacker.

Patching using UOFiddler

  1. The ultima_online_mods folder includes 00_ART_MODS_MassImport.xml (a list of item IDs and file paths for the modded art). Edit the file paths with a global “replace all” to match your system paths, or generate a custom XML using 00_mod_selector.py.
  2. Plugins > MassImport → Load XML → select 00_ART_MODS_MassImport.xml from the downloaded mods folder (loads replacement art files).
  3. Enable DirectSave, then click Start (may take ~1 minute).
  4. Your OUTPUT folder will now contain the modded .mul files.
  5. UOFiddler → UOP Packer → paste the OUTPUT folder path → turn ON Pack MUL to UOP → Start.
  6. Close UOFiddler. Copy the .mul, .idx, and .uop files from OUTPUT into your Ultima folder, overwriting existing files.
  7. Installation complete — launch Ultima Online.
  8. If your shard updates later, patch using their updater (verify), then repeat the MassImport process in UOFiddler.

Patching Troubleshooting

  • Terrain textures look mismatched/random in-game: This usually means a shard patch updated over the modded files. Re-apply mods with MassImport after updating to the shard’s latest.
  • MassImport crashes/freezes (>2 minutes): Your source .mul may be corrupted. Restore backup .mul files or delete them and verify via your shard’s launcher to download fresh unmodded versions, then re-run MassImport. This can happen after shard updates.
  • After a shard update, the client crashes with “System.OutOfMemoryException: Array dimensions exceeded supported range.” referencing ArtLoader: delete the recently updated .mul or .uop files in the client directory, verify again with the shard launcher to fetch clean copies, confirm the client launches, then run MassImport on top using UOFiddler.
